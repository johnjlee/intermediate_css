The float property

<img src="logo.png" style="float: left">

This is a way to push content to the side of the page, while still having the rest of the content wrap around it. The rest of the content wraps around the floating element and continues using the full width of the parent after it reaches the bottom of the floating element. On the surface, this looks easy to use, but as with anything in CSS, it comes with a lot of annoyances and weird corner cases.

This is what overflow:hidden is supposed to be used for:

Using overflow:hidden to fix the float problem is an abuse of the CSS spec, but it's the easiest and most common way to do it. Most people don't understand why it works.

In a nutshell, a using overflow:hidden creates a new block formatting context. Block formatting contexts cannot overlap floats. Other things (including display:inline-block) create block formatting contexts too, but they do other stuff too that makes it not suitable to fix the float problem.

One day, some developers noticed that overflow:hidden happens to create a block formatting context, but doesn't do much else. And now it's used all over the place.

http://www.w3.org/TR/CSS2/visuren.html#block-formatting

CSS annoyance #358: Containing blocks

The containing block of an element with position:absolute is not necessarily the parent element. It's the first ancestor element that has position:absolute or position:relative (... sort of).

http://www.w3.org/TR/CSS21/visudet.html#containing-block-details

One more: position:fixed

It's like position:absolute, except "containing block" means something entirely different.

CSS annoyance #1052: Centering things is a pain

You can use the position:absolute/margin:auto trick to center elements if:

  • The width/height of the inner element is explicitly set
  • You're OK with using position:absolute

This is probably the least painless way to center something vertically, so always try to do it this way if you can get away with it.

Centering horizontally with text-align

Browsers can center text no problem. If you're not trying to center text, you can tell the browser to pretend.

Careful: the text-align:center property propagates to all child elements.

Summary of centering

position:absolute with margin:auto

+ Works horizontally and vertically
- Must know width/height of child element
- Breaks normal flow

text-align:center

! Horizontal centering only
+ Works on variable-width elements
- Propagates to inner elements

line-height

! Vertical centering
+ Easiest way to center text
- Must know height of parent, but child height can be variable
- Annoying for centering non-text

span with height:100%

! Vertical centering
+ Works on variable height elements
- Need to insert extra span which can take up space
